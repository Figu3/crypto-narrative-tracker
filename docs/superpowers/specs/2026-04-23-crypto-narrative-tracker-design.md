# Crypto Narrative Mindshare Tracker вҖ” Design Spec

**Date:** 2026-04-23
**Status:** Approved design, pending implementation plan

## 1. Goal

A web app that shows the rise and fall of crypto narratives (AI agents, RWA, DeFi summer, NFTs, restaking, memes, etc.) across a zoomable timeline from 2017 to today, with a "right now" panel indicating which narratives are heating up or cooling down.

The primary user journey: open app вҶ’ see a streamgraph spanning 2017-today вҶ’ zoom into any era вҶ’ hover to inspect which narrative dominated вҶ’ glance at the sidebar for the current top 10 with вҶ‘/вҶ’/вҶ“ trend arrows.

## 2. Non-goals (explicitly out of scope for v1)

- No individual-token tracking (narratives only вҖ” tokens listed on a narrative's card for reference, not tracked).
- No paid data sources (Kaito, LunarCrush paid, Twitter API). Free-tier only.
- No real-time streaming. Weekly refresh cadence is sufficient.
- No user accounts, saved views, alerts, or notifications.
- No mobile-optimized gestures. Desktop-first; mobile just needs to be legible.
- No bump chart / heatmap toggle. Streamgraph only.
- No predictive forecast lines. Trend arrows use a simple slope heuristic, nothing modeled.
- No narrative auto-discovery. The narrative list is a hand-curated JSON file.

## 3. Architecture

Two decoupled pieces:

### A) Data pipeline
- A script (Node/TypeScript) that runs weekly via GitHub Actions.
- Fetches Google Trends for each narrative's keyword set.
- Fetches DefiLlama narrative TVL where applicable (`/categories` endpoint).
- Normalizes and writes `data/mindshare.json` (one file, all narratives Г— all periods).
- Pre-2023 data is hand-seeded once in `data/historical-seed.json` (committed, never regenerated by the pipeline).
- The pipeline merges `historical-seed.json` + freshly-fetched data into the final `mindshare.json`.

### B) Frontend
- Next.js 16 with TypeScript strict mode.
- Reads `mindshare.json` at build time (SSG) вҖ” no backend, no DB, no runtime API routes.
- Streamgraph rendered with D3 or Visx.
- Deployed to Vercel (free tier is sufficient).

### Rationale for the split
Data fetching is slow and rate-limit-prone (Google Trends). Rendering is fast and static. Keeping them separate means a failed cron does not break the live site вҖ” it just keeps serving the last good `mindshare.json` that is committed to the repo.

## 4. Data model

### `narratives.json` (hand-curated, ~25 entries, reviewed quarterly)

```json
[
  {
    "id": "ai-agents",
    "label": "AI Agents",
    "color": "#8b5cf6",
    "emergedAt": "2024-10",
    "retiredAt": null,
    "keywords": ["ai agents crypto", "virtuals protocol", "ai16z"],
    "defillamaCategory": "AI Agents",
    "relatedTokens": ["VIRTUAL", "AI16Z", "FARTCOIN"]
  },
  {
    "id": "defi-summer",
    "label": "DeFi Summer",
    "color": "#10b981",
    "emergedAt": "2020-06",
    "retiredAt": "2021-03",
    "keywords": ["defi yield farming", "yearn finance", "compound finance"]
  }
]
```

Fields:
- `id` вҖ” stable slug, used as key in `mindshare.json`.
- `label` вҖ” display name.
- `color` вҖ” stable hex color, stays consistent across the UI.
- `emergedAt` / `retiredAt` вҖ” `YYYY-MM` strings. Narratives are absent from the chart outside this window. `retiredAt: null` means still active.
- `keywords` вҖ” Google Trends query terms. Scores are averaged across the set.
- `defillamaCategory` вҖ” optional; matches a DefiLlama `/categories` entry. Missing = Google Trends only.
- `relatedTokens` вҖ” display-only reference list, not tracked.

### `historical-seed.json` (hand-seeded once, covers 2017-01 through 2022-12)

```json
{
  "ai-agents": [],
  "defi-summer": [
    { "date": "2020-06", "score": 45 },
    { "date": "2020-07", "score": 78 },
    { "date": "2020-08", "score": 92 }
  ]
}
```

Sources for seeding: Google Trends (for terms that were searched pre-2023), DefiLlama historical category TVL, news archive reading for qualitative ranking.

### `mindshare.json` (generated; one series per narrative)

```json
{
  "generatedAt": "2026-04-23T00:00:00Z",
  "series": [
    {
      "narrativeId": "ai-agents",
      "points": [
        { "date": "2024-10", "score": 12 },
        { "date": "2024-11", "score": 38 },
        { "date": "2024-12", "score": 71 }
      ]
    }
  ]
}
```

### Score definition

`score` is a 0-100 normalized mindshare value per narrative per period.

For narratives with a `defillamaCategory`:
```
score = 0.7 * googleTrendsInterest + 0.3 * defillamaTvlShare
```
- `googleTrendsInterest`: 0-100 from pytrends, averaged across the narrative's keywords.
- `defillamaTvlShare`: narrative TVL Г· total crypto TVL, rescaled to 0-100 across all narratives for that period.

For narratives without a `defillamaCategory`: `score = googleTrendsInterest`.

### Trend arrow logic (`lib/trend.ts`)

For each narrative in the "Right Now" panel:
1. Take the last 4 periods and the 4 periods before those (at the current zoom level вҖ” so "period" is monthly/weekly/daily depending on zoom).
2. Compute mean of each 4-period window.
3. Percentage change = (recent вҲ’ prior) / prior.
4. Arrow: `вҶ‘` if change > +10%, `вҶ“` if change < вҲ’10%, `вҶ’` otherwise.
5. Display the % delta next to the arrow.

### Streamgraph behavior
- X axis: time (resolution follows zoom level).
- Y axis: stacked mindshare share.
- Each ribbon = one narrative, colored with its `narratives.json` color.
- Hover: tooltip with narrative name, date, and score.
- Click a ribbon: dims others, isolates that narrative's curve; click again to restore.
- Legend below: chips to toggle individual narratives on/off.

### Zoom / range controls
- Three buttons above the chart: **Monthly** (default), **Weekly**, **Daily**.
- A mini-timeline brush below lets the user pick a date window within the selected resolution.
- Default view: monthly, full history (2017 вҶ’ now).

### Right-side "Right Now" panel
- Top 10 narratives ranked by the latest-period score.
- Each row: rank, label, arrow, % delta.
- Clicking a row highlights that narrative in the streamgraph (same behavior as clicking a ribbon).
- Retired narratives (`retiredAt` set) do not appear here.

### Footer
- "Data as of {generatedAt}" pulled from `mindshare.json`.
- Link to the source repo.

## 6. Update cadence

- **Weekly data refresh:** GitHub Actions cron runs every Monday 06:00 UTC. It executes `scripts/fetch-mindshare.ts`, writes the updated `mindshare.json`, and commits + pushes back to `main`. Vercel auto-redeploys on push.
- **Quarterly narrative review:** a calendar reminder (outside the app) to manually open `narratives.json`, retire dead narratives, add emergent ones, and adjust keywords.

## 7. Error handling and edge cases

### Data pipeline
- Google Trends rate-limits aggressively. The script uses pytrends-equivalent logic in Node (or calls out to a Python helper if needed) with a 2-second delay between requests and exponential backoff on HTTP 429.
- Per-narrative fetch failure: keep the previous period's value, append an entry to `data/fetch-errors.json`, do not crash the run.
- Full-run failure: GitHub Action fails loudly (email notification). The site continues serving the last good `mindshare.json` because it is committed.
- The script is idempotent вҖ” rerunning does not duplicate periods; it upserts by `(narrativeId, date)`.

### Data edge cases
- **Narrative with no data before `emergedAt`:** render as 0 / absent, do not extrapolate backward.
- **Retired narratives:** present in historical view, absent from "Right Now".
- **Pre-2023 gap:** `historical-seed.json` provides monthly scores; the pipeline never overwrites any period before `2023-01`.
- **Daily zoom on pre-2023 data:** fall back to the monthly value; display a small hint ("historical data is monthly-only") when the user zooms to daily on a window that crosses the 2023 boundary.

## 8. Testing

- `scripts/fetch-mindshare.ts` вҖ” unit tested with mocked Google Trends and DefiLlama responses. One integration test (gated behind an env flag) runs against the real APIs.
- `lib/trend.ts` and `lib/zoom.ts` вҖ” pure functions, unit tested with Vitest.
- `Streamgraph.tsx` вҖ” Playwright visual snapshot test against a fixed fixture dataset.
- TypeScript strict mode enforced in `tsconfig.json`. `npm run build` must pass before merging any PR.

## 9. Observability

- `data/fetch-errors.json` is committed on every cron run вҖ” a browsable history of which narratives failed when and why.
- `mindshare.json.generatedAt` is surfaced in the footer so users always know how fresh the data is.

## 10. Deployment

- Hosted on Vercel free tier.
- Repo on GitHub; weekly cron runs in GitHub Actions and pushes commits back to `main`, which triggers a Vercel redeploy.
- No environment secrets needed for v1 (all data sources are public/unauthenticated).
